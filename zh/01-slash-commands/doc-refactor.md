# 檔案重構

根據專案型別重構專案檔案結構：

1. **分析專案**：識別型別（庫/API/Web 應用/CLI/微服務）、架構和使用者角色
2. **集中管理檔案**：將技術檔案遷移到 `docs/`，並保留正確的交叉引用
3. **整理根目錄 README.md**：將其精簡為入口頁，包含概覽、快速開始、模組/元件摘要、許可證和聯絡方式
4. **為元件補充檔案**：新增模組/包/服務級別的 README 檔案，包含安裝和測試說明
5. **按主題組織 `docs/`**：
   - 架構、API Reference、資料庫、設計、故障排查、部署、貢獻（根據專案需要調整）
6. **建立指南**（按需選擇）：
   - 使用者指南：面向應用終端使用者的檔案
   - API 檔案：API 的端點、認證和示例
   - 開發指南：環境搭建、測試、貢獻流程
   - 部署指南：服務/應用的生產部署
7. **所有圖表都使用 Mermaid**（架構圖、流程圖、Schema 圖）

保持檔案簡潔、易掃讀，並與專案型別保持上下文一致。
