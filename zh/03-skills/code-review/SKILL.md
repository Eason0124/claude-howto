# 程式碼審查 Skill

這個 skill 提供全面的程式碼審查能力，重點關注：

1. **安全分析**
   - 身份驗證 / 授權問題
   - 資料洩露風險
   - 注入漏洞
   - 密碼學弱點
   - 敏感資料日誌記錄

2. **效能審查**
   - 演算法效率（Big O 分析）
   - 記憶體最佳化
   - 資料庫查詢最佳化
   - 快取機會
   - 併發問題

3. **程式碼質量**
   - SOLID 原則
   - 設計模式
   - 命名規範
   - 檔案
   - 測試覆蓋率

4. **可維護性**
   - 程式碼可讀性
   - 函式長度（建議少於 50 行）
   - 圈複雜度
   - 依賴管理
   - 型別安全

## 審查模板

對每一段被審查的程式碼，請提供：

### 摘要
- 整體質量評分（1-5）
- 關鍵發現數量
- 建議優先關注的區域

### 關鍵問題（如有）
- **問題**：清晰描述
- **位置**：檔案和行號
- **影響**：為什麼這很重要
- **嚴重性**：Critical / High / Medium
- **修復**：程式碼示例

### 按類別列出發現

#### 安全性（如有問題）
列出安全漏洞及示例

#### 效能（如有問題）
列出效能問題，並說明複雜度

#### 質量（如有問題）
列出程式碼質量問題，並給出重構建議

#### 可維護性（如有問題）
列出可維護性問題，並給出改進建議
