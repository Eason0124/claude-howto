# 貢獻者行為準則

## 我們的承諾

我們致力於為所有人提供一個熱情且鼓舞人心的社群。我們承諾，參與本社群的每個人都應擁有免受騷擾的體驗，不論其年齡、身體大小、種姓、殘障、族裔、性別特徵、性別認同與表達、經驗水平、教育程度、社會經濟地位、國籍、外貌、種族、宗教，或性取向與性身份如何。

我們致力於打造一個積極、包容且安全的環境，讓所有貢獻者都感到被尊重、被重視，並能充分發揮自己的能力。

## 我們的標準

有助於營造積極環境的行為包括：

### 保持尊重
- 使用熱情且包容的語言
- 尊重不同的觀點、立場和經歷
- 愉快地接受建設性批評
- 尊重文化和語言差異
- 認可並慶祝多元背景

### 保持協作
- 共同解決衝突
- 給予他人應得的署名和認可
- 幫助他人學習和成長
- 分享知識並積極提問
- 支援社群裡的其他成員

### 保持專業
- 保持討論聚焦且高效
- 避免貶低或侮辱性言論
- 在 issue 和 discussion 中保持主題相關
- 使用清晰且尊重的溝通方式
- 假設他人的訊息都出於善意

### 保持包容
- 歡迎新貢獻者
- 幫助剛接觸專案的人上手
- 提供建設性的反饋
- 接納不同視角的人
- 為安靜的聲音創造被聽見的空間

## 不可接受的行為

以下行為被視為騷擾，在本社群中不可接受：

### 騷擾與歧視
- 與性別、性別認同與表達、性取向、殘障、心理疾病、神經型別、外貌、體型、年齡、種族或宗教相關的冒犯性言論
- 關於某人生活方式或生活選擇的不受歡迎評論
- 故意使用錯誤稱謂，或使用“死名”或被拒絕的名字
- 與主題無關的露骨性影象或行為
- 未經同意或在要求停止後仍進行肢體接觸或模擬肢體接觸

### 濫用行為
- 指向他人的暴力威脅或暴力言論
- 唆使暴力或針對個人的暴力言論
- 故意恐嚇
- 跟蹤或尾隨
- 反覆聯絡造成騷擾
- 任何其他合理上可視為不當的行為

### 線上騷擾
- 持續幹擾線上討論
- 釋出他人隱私資訊（人肉搜尋 / doxxing）
- 不受歡迎的性關注或性騷擾
- 刷屏、挑釁或故意搗亂
- 垃圾廣告或重複性宣傳
- 散佈錯誤資訊或虛假資訊

### 報復
- 因他人舉報違規而採取報復行動
- 因他人舉報不當行為而區別對待
- 以舉報作為威脅

## 適用範圍

本行為準則適用於：
- 所有社群空間（GitHub issues、pull requests、discussions 等）
- 專案倉庫和檔案
- 由專案組織或代表專案參加的活動和聚會
- 代表專案或社群的公共場合
- 與專案事務相關的私人溝通

本行為準則適用於所有專案空間；當個人在公共場合代表專案或社群時，也同樣適用。代表專案或社群的例子包括使用官方專案郵箱、透過官方社交賬號發帖，或線上上/線下活動中擔任指定代表。

## 執行

社群負責人負責澄清並執行可接受行為標準，並會針對任何被認為不當、具有威脅性、冒犯性或有害的行為採取適當且公正的糾正措施。

### 舉報違規

如果你經歷或目睹了不可接受的行為，或者有其他擔憂，請透過以下方式報告：

1. **Email**：聯絡專案維護者（如可用，請填寫聯絡方式）
2. **GitHub Issues**：使用私密舉報功能（如果你的 GitHub 設定中可用）
3. **直接訊息**：直接聯絡專案維護者

請提供：
- 你的聯絡方式
- 涉及人員的姓名（如果你願意提供）
- 事件描述
- 上下文和相關連結
- 事件對你的影響

**所有報告都會被保密。**

### 調查與響應

社群負責人將：
- 及時且公正地審查和調查所有投訴
- 對舉報人保持保密
- 向舉報人反饋處理進展
- 根據調查結果採取適當行動

可能的措施包括：
- 私下警告或輔導
- 公開道歉（若違規者同意）
- 暫時或永久移出社群空間
- 取消貢獻者許可權
- 必要時升級至 GitHub 或其他平臺

### 申訴流程

如果你認為某項決定有誤，可以透過以下方式申訴：
1. 提供新的資訊或上下文
2. 請求多位維護者複核
3. 在初次決定後的 30 天內跟進

## 如何尊重地表達分歧

技術社群中出現分歧是正常的。以下是尊重地表達不同意見的方法：

### 應該做的
- ✅ 聚焦觀點，而不是人
- ✅ 詢問澄清問題
- ✅ 承認對立觀點中的合理之處
- ✅ 使用 “我” 陳述（例如：“我不同意，因為……”）
- ✅ 引用來源和證據
- ✅ 感謝他人的觀點

### 不應該做的
- ❌ 人身攻擊
- ❌ 使用貶低性語言（如“這太蠢了”）
- ❌ 預設他人惡意
- ❌ 提及無關的過往衝突
- ❌ 升級語氣或變得敵對
- ❌ 故意歪曲他人觀點

### 尊重分歧示例
```
❌ 不好：“這是個愚蠢的想法，你顯然不瞭解這個專案。”

✅ 更好：“我理解你的意思，但我擔心這種做法在更大的專案裡可能
不太能擴充套件。你考慮過這個基準測試裡展示的效能影響嗎？我很想聽聽
你的看法。”
```

## 搭建橋樑

我們相信對話與和解的力量：

- 如果別人指出你的行為問題，請認真傾聽並反思
- 如果你造成了傷害，願意道歉
- 支援他人的學習和成長
- 記住每個人都會犯錯
- 聚焦解決方案，而不是歸咎
- 在堅持責任的同時，保持善意

## 遵守本準則的好處

透過維護一個尊重且包容的社群，我們能夠：
- 吸引擁有不同視角的貢獻者
- 透過協作產出更好的解決方案
- 建立更牢固的關係
- 讓專案對新成員更友好
- 產出更高質量的檔案和示例
- 培養學習與成長的文化

## 特別說明

### 對維護者
- 以身作則，遵守本準則
- 對舉報保持響應
- 在決策上保持透明
- 幫助公正地解決衝突
- 為所有人創造安全環境

### 對貢獻者
- 假設維護者和其他貢獻者都懷有善意
- 提供建設性的反饋
- 支援新成員
- 幫助維護社群標準
- 對社群文化承擔責任

### 對所有人
- 記住每條訊息背後都是一個人
- 如果感到沮喪，先暫停一下
- 需要支援時請聯絡社群負責人
- 幫助創造你希望參與的社群
