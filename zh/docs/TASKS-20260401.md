## 第 1 階段 - 基礎設施與薄弱模組（2026 年 4 月）

**目標里程碑：M1**

### 支柱 4：新手上手

- [ ] **T-001** - 編寫 `scripts/quickstart.sh`
  - 檢測 `~/.claude/` 目錄（如缺失，則在使用者確認後建立）
  - 將第一個 slash command + CLAUDE.md + skill 複製到使用者配置中
  - 在 CLAUDE.md 末尾追加 agent-discovery 行，連結到 AGENT-INDEX.md
  - 冪等：跳過已有檔案並給出警告，絕不覆蓋
  - 完成後列印下一步

- [ ] **T-002** - 建立 `QUICKSTART.md`（前 15 分鐘視覺化指南）
  - 使用程式碼塊記錄帶註釋的終端步驟（儘量避免 PNG 截圖；優先 ASCII 或 Mermaid）
  - 記錄每條命令對應哪個視覺結果，方便後續重新採集

- [ ] **T-003** - 為所有 10 個模組的 README 新增難度徽章
  - 格式：`> 🟢 **Beginner** | ⏱ 30 minutes`，放在每個模組 README 頂部
  - 在徽章下方新增 "What you'll build" 的要點預覽

### 支柱 7：活課程體系

- [ ] **T-004** - 建立 `WHATS-NEW.md`
  - 格式：`## CC vX.X — DATE`，每個功能變更使用專案符號
  - 至少新增一條當前 CC 版本的記錄

- [ ] **T-005** - 為所有 10 個模組的 README 新增版本徽章
  - 格式：`> ✅ Verified against Claude Code **vX.X.X** · Last verified: YYYY-MM-DD`
  - 在每個模組的 frontmatter 中加入 `cc_version_verified`

- [ ] **T-006** - 建立 `.github/workflows/staleness-check.yml`
  - 排程：每週一 09:00 UTC
  - 從 frontmatter 讀取每個模組的 `last_verified` 日期（使用 `yq` 或 10 行 Python 指令碼）
  - 對 30 天以上未驗證的模組建立 GitHub Issue
  - 如果已有同名 open issue，則跳過

### 支柱 6：內容質量（CLAUDE.md）

- [ ] **T-007** - 為專案建立根目錄 `CLAUDE.md`
  - 作為專案自身配置，展示最佳實踐
  - 記錄貢獻規範、模組結構和維護流程

### 支柱 2：AI 代理索引（生成器）

- [ ] **T-008** - 編寫 `scripts/build-agent-index.py`
  - 讀取全部 10 個模組 README
  - 提取：標題（→ topics）、程式碼塊（→ 帶語言標籤的示例）、表格（→ 參考資料）
  - 輸出 `agent-manifest.json`，schema 按 CEO 計劃定義
  - 輸出 `AGENT-INDEX.md`（人類可讀摘要）
  - 校驗引用是否指向真實檔案

### 模組 08-checkpoints：完整深度處理（311 → 800+ 行）

- [ ] **T-009** - 用真實場景重寫模組導語
  - 示例："你的重構把 3 個東西弄壞了。checkpoint 如何幫你恢復..."

- [ ] **T-010** - 新增 checkpoint 策略決策樹（Mermaid flowchart）
  - "我想做 X" → 順著箭頭走 → 到達對應 checkpoint 模式

- [ ] **T-011** - 新增 3 個工作流模板
  - 安全重構工作流
  - 使用 checkpoints 做 A/B 測試
  - 災難恢復工作流

- [ ] **T-012** - 新增與 planning mode 的整合部分

- [ ] **T-013** - 新增 checkpoint 生命週期的 Mermaid 圖

- [ ] **T-014** - 新增 2 個以上 "Try It Now" 模組，並提供可驗證命令
  - 每個模組：上下文句、要執行的命令、預期輸出說明

- [ ] **T-015** - 新增 "Patterns & Recipes" 部分，包含 2 個以上命名模式
  - 每個模式：問題、解決方案程式碼、適用場景、不適用場景

### 模組 06-hooks：完整深度處理

- [ ] **T-016** - 用真實場景重寫模組導語
  - 示例："你的 CI 在 20 分鐘後才發現 lint 錯誤。hooks 2 秒內就能抓住..."

- [ ] **T-017** - 新增 hooks 決策樹（Mermaid flowchart）
  - "我想在 Y 發生時觸發 X" → 順著箭頭走 → 到達對應 hook 型別

- [ ] **T-018** - 新增 5 個完整 hook 示例，帶完整指令碼（每個約 80 行）
  1. 寫入時自動格式化（prettier/black）
  2. 提交時安全掃描（trufflehog/bandit）
  3. 檔案變更時執行測試
  4. 會話結束時傳送通知
  5. 提示詞校驗（阻止危險模式）

- [ ] **T-019** - 新增 2 個以上 "Try It Now" 模組，並提供可驗證命令

- [ ] **T-020** - 新增 "Patterns & Recipes" 部分，包含 2 個以上命名模式

**M1 檢查點：基礎設施上線，2 個最薄弱模組完成深度升級**
